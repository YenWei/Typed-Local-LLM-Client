## Package Structure

### `llm_client/exceptions.py`
Custom exception hierarchy:
- `LLMErrorBase`
- `LLMUnavailableError`
- `LLMTimeoutError`
- `LLMValidationError`

### `llm_client/schemas.py`
Current schema models:
- `LLMRequest`
- `LLMResponse`
- `TaskSummary` as the current example output schema

### `llm_client/retry.py`
Current retry behavior:
- generate with internal helper
- extract JSON from response text
- validate extracted JSON against the requested output schema
- retry on validation-related failures
- on first JSON failure, appends a stricter JSON-only correction instruction
- on first schema mismatch, appends a stricter schema-matching correction instruction

### `llm_client/client.py`
Intended public interface:
- `ping()`
- `generate_reply(request, output_Type)`

Internal helpers kept for testing / internal execution flow:
- `_generate()`
- `_generateRAW()`

### `tests/`
Coverage exists for:
- schema validation
- retry behavior
- client ping / basic generation
- typed `generate_reply()` success and failure behavior
- integration test against live Ollama host

## Important Design Decisions
- Public interface should stay small: `ping()` and `generate_reply()` only
- underscore-prefixed methods are intentionally internal helpers, not public API
- integration test should not hardcode one machine-specific host
- benchmark evidence is kept separately from unit tests
- `LLMResponse` is a stable wrapper; typed validated payload lives in `LLMResponse.data`
- output schema is passed into `generate_reply()` so one wrapper can support multiple typed response contracts

## Known Caveats
- The original portfolio plan assumed a slightly richer Ollama request/structured-output story than what was straightforward in the handwritten implementation.
- `benchmark.py` works as an experiment driver, but is still rough compared with the quality of the captured benchmark results in `benchmark.txt`.
- `TaskSummary` is only a starter schema used to prove the typed pipeline; it is not necessarily the final most meaningful schema for the portfolio.
